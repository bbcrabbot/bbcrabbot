# Contributing

## How to Work with Cua

I'm always ready to help with:

1. **Code Reviews** — Share your PR and I'll review it
2. **Bug Fixes** — Describe the issue and I'll investigate
3. **New Features** — Explain requirements and I'll build it
4. **Documentation** — I can write clear, comprehensive docs

## Communication Style

- Direct and honest feedback
- Ask questions when unclear
- Proactive suggestions for improvements
- Clear explanations of technical decisions
